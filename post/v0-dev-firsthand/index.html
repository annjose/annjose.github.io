Vercel's v0.dev: My Firsthand Experience

An introduction, hands-on experience and thoughts on why it matters to everyone

04 Nov 2024
how-to / web-development

I have been using Vercel products for my web projects - their Next.js app framework, their deployment infrastructure and their AI SDK. All these tools are easy to use and onboard, they are reliable and fast - and most of them are open source, which I love ‚ù§Ô∏è.

Their latest innovation is v0.dev, an AI-powered tool that allows building frontend applications using a conversational chat interface. When I first heard about it, I thought it would be just another AI coding tool with a chat interface, similar to asking ChatGPT or Claude to generate code.

But recently, Vercel announced a major update to v0.dev that includes integration with the popular shadcn UI components and external npm packages. So I decided to try it out yesterday.

I was blown away by the experience. In just an hour, I was able to build a functional, good-looking website using v0.dev’s chat interface. It was like magic - a series of “jaw-drop” moments. I was so excited that when George got home from work, I ran downstairs to rave about how awesome the tool was. Fun fact - he had already tried it out and was thoroughly impressed, just that he admired it in a more quiet way than my excited reaction üôÇ. Now it it time to share it with you.

In this post, I’ll share my firsthand experience with v0.dev - what I loved about it, a few quirks I found while using it, and what I wish the tool could do better. Most importantly, I’ll discuss what I believe v0.dev means for developers, designers, entrepreneurs, and anyone building for the web.

If you read this much and are thinking this is some crazy tech stuff and it’s not relevant to me, I urge you to read the last section Final Thoughts - because there I explain how (and this may sound clich√©) this tool can transform your own abilities (or lack thereof) to build any website and do things you couldn’t imagine before. Perhaps that will convince you to come back here and read the next section that explains what v0.dev is and what you can do with it.

What is v0.dev?

v0.dev is a generative UI system that can generate code using modern web technologies like Next.js, React and Tailwind. In the latest update, they also revamped the conversational UI to be more meaningful and added the ability to pull third party packages into the generated code.

  • v0 can generate code in terms of UI, Next.js pages, or plain TypeScript / JavaScript. You can create a first version of the page and then ‘modify’ it through the prompts in the chat interface. For example, you can ask ‘show me another version of this page’, ’the blue is too dark, make it lighter’.

  • In addition to the individual pages and components it creates, v0 can create an end-to-end project with all the files in the appropriate folder structure as per the Next.js convention. This was my biggest surprise. I asked it to create a charity website and asked it to create a few pages (About, Contact etc.). Then I clicked the button Add to codebase and it gives a npx command with a git repository. I copied it and ran it on my iTerm. And it created the project. That was awesome!

    Ideally, you should be able to run the app using npm run dev. But it didn’t work for me right out of the box, there were a few build issues that I had to fix - more on that later.

  • You can give a website’s URL or screenshot and ask it to recreate the website using Next.js, Tailwind. I tried this with my website and it was fun watching it generate the code - that too with an animation as if it ‘printing’ my website.

  • The UI code that it generates is built using shadcn UI components, a popular collection of re-usable components that are accessible, customizable and open source. The reverse is also true, every component on shadcn.ui is editable on v0. So you can choose any shadcn component, for example, this calendar component and click the button ‘Open in v0’ in the Preview mode. It will open up v0 with that component. So you can edit and customize it through the chat interface. This two-way integration works pretty well.

How is it different from ChatGPT or Claude?

It uses the up-to-date information of Next.js, including things that are in theit canary or experimental stream. This is the #1 thing that differentiates v0.dev from ChatGPT or Claude. We know that these tools also offer a conversation UI to answer questions, show code and execute it. But this is fine-tuned to the latest information. For example, here is the output of the same question to all these tools - it gave the client side form submission instead of the Next.js best practice of server actions. it also gives answers based on the Pages router instead of the newer App router paradigm. Interestingly, App router is no longer new, it is the defacto way of building Next.js apps.

Other goodies

  • it can use third-party npm packages to render the UI. For example, it uses the use-debounce package to show confettit at the end of a form submission interaction. It used three.js library and generates complex code to render grass.

  • it can generate diagrams using Mermaid to respond your questions with answers that are more meaningful with diagrams.

  • You can publish your code and the prompts with anyone.

  • In addition to Next.js, it can generate code in Vue.js. It doesn’t support other web frameworks like Svelte or Flutter for mobile. In this video by Lee Robinson, they say that they are actively working on adding support for more frameworks.

My experience of using v0

I used v0 for a few hours yesterday - at first to just create random projects like hello world, to do list etc. But then I decided to use it to create a charity website that I wanted to create for my uncle. I had been thinking about what this website should contain, referring to similar websites as inspiration and had selected a couple of templates to use. I also had a bunch of notes from talking to my uncle about all the work that his organization does.

I just started giving all this information as ‘requirements’ to v0. At first, I typed out the snippets of information and asked it to create a page. After that, I found it easier to just dictate the instructions, so I took my hand off the keyboard and started the dictation mode and just started narrating what I want in the website. v0 diligently followed the instructions and created the pages, tweaked it as I asked it to. At the end, I asked it to create a project ‘Add to codebase’

The joy and magic

It was a joy to start with

  • I gave all the prompts through voice. Didn’t touch the keyboard for the most part. it describes what it is going to do and then shows the results. This chain of thought reasoning is very useful because if it gets something wrong, you can see where it went wrong.

The hiccups along the way

When you try to run npm run dev you get an error like this:

This is because of a name conflict in the project

You have two entities named Page in the same scope, which is causing confusion for the compiler. Here’s how to resolve this issue: The imported Page component:

1

import { Page } from "@/components/page"

The default exported function also named Page:

1
2
3

export default function Page() {
  return <Page />
}

The best way to solve it is to address the root cause - there should be only one Page component for the home page. So rename the page.tsx in components to HomePage.tsx and modify the main Page.tsx to refer to it as follows:

1
2
3

export default function Page() {
  return <HomePage />
}

Note that you need to put the feault keyboard on the export statement

When i asked it create multiple pages, the exported project had only one page. I expected all the three pages to be added to the project. So I had to manually copy them

Another error was Cannot find module ‘@/components/ui/use-toast’ or its corresponding type declarations.ts(2307)

from the line import { toast } from “@/components/ui/use-toast”

Here, the use-toast is a hook which is in the folder `/components/hooks so update the path in the import statement import { toast } from “@/hooks/use-toast”

Then you have one more error Cannot find module ‘@/components/ui/toast’ or its corresponding type declarations.ts(2307)

from the use-toast hook file. So you need to add the shadcn component for this

1

npx shadcn@latest add toast

Refer https://ui.shadcn.com/docs/components/toast for details on this component

My wishlist for v0.dev

While my initial experience with v0 was fantastic, there are a few things that I didn’t quite like and would give as feedback to the v0 team.

  1. When it is asked to create multiple pages, the project should contain all of them, not just the page that was in the selection.

  2. I want continuity of the project. Once I create the project, there is no link between the chat and the project. It would be great if there is a way to carry it back and forth between the IDE and the chat. Perhaps a seamless handoff to Cusror AI or GitHub Copilot would be awesome.
